FlxSprite.makeGraphic not working as expected

SoulBeaver, 3 months 1 week ago

I'm currently just experimenting with HaxeFlixel after downloading flixel 3.0.0-dev from Github a few days ago. I tried creating a small, gray box the width of the screen, but the output is not what I expected.

This is the working code:

_menuBar = new FlxSprite(0, FlxG.height - 60); _menuBar.makeGraphic(FlxG.width, 60, FlxColor.GRAY); add(_menuBar);

I don't know why this works. In the configuration xml I changed the height and width of the program to 1366x768. I'm expecting a very tiny, 60px tall, gray box. Yet I have this:

When I try to use the height I intended, a 200px tall gray box:

_menuBar = new FlxSprite(0, FlxG.height - 200); _menuBar.makeGraphic(FlxG.width, 200, FlxColor.GRAY); add(_menuBar);

Can anybody tell me why the output is as it is?


Garfunkel
3 months 1 week ago

The FlxGame height seems to be about 240. It can be seperate from your window size, the 1366 * 768 you set.

So it depends on how you initialized FlxGame.

The default template contains this code in ProjectClass.hx:

var stageWidth:Int = Lib.current.stage.stageWidth;
var stageHeight:Int = Lib.current.stage.stageHeight;
var ratioX:Float = stageWidth / 640;
var ratioY:Float = stageHeight / 480;
var ratio:Float = Math.min(ratioX, ratioY);
super(Math.ceil(stageWidth / ratio), Math.ceil(stageHeight / ratio), MenuState, ratio, 30, 30);

Meaning flixel scales itself to the size you set in the xml. As its base size it uses 640 * 480. So take a look at where you call the FlxGame constructor.
