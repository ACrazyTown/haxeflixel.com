So last night I ported my game from 2.0.0-alpha3 to the latest version of Haxeflixel and after making all the needed changes (package name changes, animation changes, etc) I was able to get my game to compile and run. However, I noticed that it seemed like nothing was lining up correctly. I took this screenshot with the debugger on and it shows the issue pretty clearly. Any idea why simply moving from 2.0 to 3.0 would cause this? Is there some offset I need to reset or change?

I haven't looked into the issue deeply, but I was wondering if anyone else has had a similar issue after upgrading.

Okay I figured out the issue. I'm not sure at all what is really causing it behind the scenes, but I have some speculation. It took some trial and error, but it came down to just a single line.

 

override public function create():Void { currentTileDisplay = new FlxSprite(1, 1); // If this graphic is set to the same as the TileMap being used AND set before the TileMap is it offsets the tiles. // So this loadGraphic call can be added below the call to new Level() or the image needs to be changed. currentTileDisplay.loadGraphic("assets/images/PlatformTileset.png", false, false, 16, 16); currentTileDisplay.animation.frameIndex = currentTile; currentTileDisplay.visible = true; currentTileDisplay.scrollFactor = new FlxPoint(0,0); add(currentTileDisplay); add(new Level("world1.csv", "assets/images/PlatformTileset.png")); }

So apparently the issue is that I was using the same asset/image to as a sprite display on the screen that I use as the tileset for the level's map. Somehow this is causing the issue. I can only imagine it has to do with Flixel's internal handling of assets?

 

Here is a gist of the platformState and the Level class. Nothing else in the project has been changed, so I figured it made sense to only upload the relevant files. https://gist.github.com/jah2488/7539289
