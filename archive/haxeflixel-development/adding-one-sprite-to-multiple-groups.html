Adding one sprite to multiple groups

Tripod, 1 month 1 week ago

Recently Beeblerox helped me to find a strange bug, I had 2 similar sprites (same code), but one sprite had framerate two times than the other(animation was moving faster), the bug was that when I added a special group for those sprites, I removed only one from PlayState, other was added to the 2 groups.

So the question is how can we automatically be warned about such things? My idea was that FlxSprite to be self aware of were it was added and if you try to add to another group without removing it from the first one it should throw an exception, but Beeblerox told me that there are valid use cases of the same sprite being in two different groups, for example for processing colisions, so is there another way to solve this problem?

Example code:

class PlayState extends FlxState { override public function create():Void { var sprite1 = new MySprite(10, 10); var sprite2 = new MySprite(50, 10); add(sprite1); var group = new FlxGroup(); group.add(sprite1); group.add(sprite2); add(group); } }

In this example, sprite1 will have the animation speed 2 times of sprite2 because it was both added to group and to PlayState.

This type of bug is caused by bad programmer behavior, it's not really a problem of the engine. You can add a sprite to multiple groups, but if you're not careful about ensuring those groups are not added to your FlxState, you will run into issues like this.

Your suggestion of notifying the user if they add the same sprite to multiple groups is possible, but it would add a some performance cost when adding any sprite to an FlxGroup, which is not desired.

crazysam

It still wouldn't be ideal, since the update call of the sprite in multiple groups would still get called multiple times, which would cause it to move at double its speed if it had a velocity.

Whatever solution we add to this would have to be preventative and debug only. Like checking every group within FlxState to see if a sprite is already added to one (and print a warning to the log if it is).

We can't "fix" this problem, we can only warn users about it.
