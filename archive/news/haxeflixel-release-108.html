HaxeFlixel Release 1.08

Chris, 11 months 3 weeks ago

Just in time before the new year we have HaxeFlixel Release 1.08. Thanks to the dedicated effort of Beeblerox we have updates and a refactor of the cpp draw order stack that makes the api for mobile/desktop targets easier to use and understand.

 

You no longer need to write any layer code as descibed here. To see the new changes in action please see the demos at https://github.com/Beeblerox/HaxeFlixelDemoes. Basically if you are targettnig mobile/desktop you will need to specify texture atlas' so that you can optimise the number of draw calls for the gpu. Texture Atlas' are a common technique when targetting gpu that AS3 Flixel has never been able to do in the past.

 

With the wonderful updates to NME 3.5.x please update your flixel and nme using haxelib command to update their latest versions.

 

haxelib upgrade

 

There is only one conflict in the current NME 3.5.1 with some changes in the asset type in the nmml parser. If you encounter any compiler errors please check this post. I have seen a commit in the nme svn that should fix this in the next nme release. 

 

This website continues to have a helpful and positive community. We all look forward to an even more exciting new year with more games and enhancements made for HaxeFlixel and the larger Haxe community.

 

Happy New Year!

 

 

Changelog - v1.08
------------------------------
* New draw stack rendering system for cpp and neko targets (replacement for layer system). It is simpler to use but little bit slower.
* NME 3.5.1 compatible
* Bug fixes for FlxText (shadow problem) and tweening (for oneshot tween)
* FlxTextField class now works on flash target also and can be used as input field, but I've removed multicamera support
* Small optimizations from @crazysam
* Added FlxBackdrop addon class
* Added TaskManager addon (ported from AntHill: https://github.com/AntKarlov/Anthill-Framework)
* Added onFocusLost() and onFocus() methods to FlxState class. These methods will called when app losts and gets focus. Override them in subclasses.
