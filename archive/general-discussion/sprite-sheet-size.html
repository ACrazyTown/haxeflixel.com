You can't use TexturePacker with HaxeFlixel right now (i will work on this feature after next release), but you can combine images in one big atlas at runtime. Here is a little doc about this, plus as usage example you can use BunnyMark demo code

Packing images into atlases is very effective for mobile targets since it could dramatically decrease the number of drawcalls.

And there is no need to have spritesheets with dimensions of power of two - NME is very "smart" about this and will handle it for you.

goshki

Just a heads-up: mind the size of a sprite-sheet on mobile targets.

 

For Santa's Christmas Presents (http://j.mp/santas-christmas-presents) I've created a sprite-sheet of presents (in front of Santas) containing 8 images. Also, I've provided all the graphics in several sizes so that bigger screens would get better quality graphics. The highest resolution sprite-sheet had dimensions of 256x2048 and only after release did I find that on Google Nexus 4 the presents were not displaying at all because the sprite-sheet size was too big for the GPU. All other graphics were displaying correctly. I've solved this problem by splitting the sprite-sheet into separate files.
