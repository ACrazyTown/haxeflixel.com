Hey Sam, I don't know if you've taken a look at this yet, but I spent a decent amount of time looking into it and I have a couple of observations:

1. FlxTypedButton - There are issues in a few places in here because the touches don't take into consideration what touch ID the touch events are for. And because the update() method actually wires itself up for the TOUCH_END events (which I think is a mistake to do so since the FlxTouchManager already handles it), any touch end event will essentially trigger the onMouseUp method in FlxTypedButton of EVERY button, since they don't keep track of the touch point ID.

2. FlxTouchManager - The handleTouchBegin method, when reusing a touch from the cache, doesn't update the touch position. As a result, if a touch was just released and very quickly another touch occurs at a different location, the new TouchEvent has the same touch point ID and as a result the same touch instance from the cache is now being re-used. However, the touch still thinks its at the previous touch's location, so the wrong button press occurs.

I've made some decent changes to both classes to get things to a point where I think it's much better, but I still periodically see some odd behavior, which could just be from my inexperience with the libraries. I could issue a pull request on the repo if you're interested in seeing what I did.
