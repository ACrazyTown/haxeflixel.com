Yes. Add all the sortable sprites to a group, and call that function each update. You can set any variable that you want to sort by and in what direction.

If you have foreground layers (layers that allways will be drawn over the sprites) just add it after the sprites.

override public function create():Void { // create sprites, add to groups, etc. add(background); add(spritesGroup); add(foreground); } override public function update():Void { spritesGroup.sort("y", FlxTypedGroup.ASCENDING); }

crazysam
2 months 3 weeks ago

The idea is that each sprite would change which group they're stored in (since this is what governs draw order), instead of sorting the groups, the sprites  would sort themselves into the correct group. For example:

// in Sprite class var foregroundGroup:FlxGroup; var midlayerGroup:FlxGroup; var backgroundGroup:FlxGroup; var _container:FlxGroup; override public function update() { super.update(); if(y <= 0.25 && _container != backgroundGroup) { swapContainer(backgroundGroup); } else if(y <= 0.5 && _container != midlayerGroup) { swapContainer(midlayerGroup); } else if(y <= 0.75 && _container != foregroundGroup) { swapContainer(foregroundGroup); } } private function swapContainer(newContainer:FlxGroup) { if(_container != null) _container.remove(this); newContainer.add(this); }

Ok, a better way to do the sorting. First you need to set the group like this

var spriteGroup:FlxTypedGroup<FlxSprite>; override public function create():Void { // FlxGroup extends FlxTypedGroup<FlxBasic> so it won't work because FlxBasic doesn't hold position spriteGroup = new FlxTypedGroup<FlxSprite>(); // add sprites to the group } // Override draw and do the sorting here because update() will be called more than draw() override public function draw():Void { spriteGroup.members.sort(mySort); super.draw(); } // The function that will sort the sprites private function mySort(a:FlxSprite, b:FlxSprite):Int { if (a.y > b.y) { return 1; } else if (a.y < b.y) { return -1; } else { return 0; } }

I'm not sure about overriding draw() but it does the trick, maybe there is a better method.

This method is better than the sorting method I posted before. In my tests with the FlxBunnyMark demo and 500 bunnies it works at ~60fps on neko, windows and html5. Sadly on flash it works at ~30-35fps but it's still way better than the other method that gives ~9-10fps XD
