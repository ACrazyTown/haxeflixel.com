I guess this depends on what exactly you want - instant tile-to-tile movement with each key press, or a transition "animation" between? Tweening would only really make sense for the latter. The former could be accomplished fairly easily with a simple timer (FlxTimer, or just a var that counts the frames) and coordinate manipulation.

Here's an example:

private var MOVEMENT_SPEED:Int = 4; private var TILE_SIZE:Int = 8; private var _keyFramesPressed:Array<Int> ; override public function update():Void { attemptMove(["LEFT", "A"], true, -TILE_SIZE, 0); attemptMove(["RIGHT", "D"], true, TILE_SIZE, 1); attemptMove(["UP", "W"], false, -TILE_SIZE, 2); attemptMove(["DOWN", "S"], false, TILE_SIZE, 3); super.update(); } private function attemptMove(KeyArray:Array<String>, Horizontally:Bool, Amount:Int, Direction:Int):Void { if (FlxG.keyboard.anyPressed(KeyArray)) { _keyFramesPressed[Direction]++; if (_keyFramesPressed[Direction] >= MOVEMENT_SPEED) { if (Horizontally) { _player.x += Amount; } else { _player.y += Amount; } _keyFramesPressed[Direction] = 0; } } else { _keyFramesPressed[Direction] = 0; } }

Admittedly, the logic I came up with here is probably needlessly complex / confusing. Basically, there's an array that stores the amount of frames an individual key has been pressed - this time is reset when the key is no longer pressed or the player is moved. I find it handy to use the functions within FlxG.keyboard to check against multiple keys at the same time, this way you can support multiple control styles like WASD and arrow keys in this example with very few code.

Regarding FlxTween, this demo should help. I'm sure you can find a demo for Actuate as well.
