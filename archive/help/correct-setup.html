Hello hero_bash, 

Is the haxelib flixel out of date(2.0 alpha) and should I use the v3 found in the github?

 The flixel on haxelib was the last release and the general feedback from that release was positive. After this all the core developers decided to refactor and improve the API itself, so the 3.x version currently on dev branch is very different. It is in the final stages of testing and bugfixing. Since they are so different and the release isnt too far maybe it would be good to start off learning the new api. Best way is from the github.com/haxeflixel/flixel-demos repositories, they have been updated to work with dev. Docs for the new version are now going to be in git also they are a wip feedback from someone who is new would be appreciated https://github.com/HaxeFlixel/haxeflixel.com/tree/master/src/documents/d.... All the best let us know how you go ;)

For samples if you mean the demos checkout the readme about the compatible git tag release for the haxelib flixel version. The "org" package is removed in HaxeFlixel 3.x .

Haxelib lets you use git repositories with a haxelib.json file you can see how to install the latest dev version here https://github.com/HaxeFlixel/haxeflixel.com/blob/master/src/documents/d...

I use Sublime and Intellij for OpenFL projects, but I am aware other devs here use the latest development builds of Flash Develop with HaxeFlixel.
