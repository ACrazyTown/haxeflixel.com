What Replaced goToAndStop?

ezacharyk, 1 month 2 days ago

In my current game project, I have a dice animation that I use when picking random values. It is a simple six-sided  style animation. Prior to updating to 3.0.0, I was using a simple set up of the play function followed by a call to goToAndStop passing in the frame that was selected at random.

After the update to 3.0.0, there is no goToAndStop function in the FlxAnimationController class. 

Is there an alternative that I should be using now?

Here is my existing animation code:

private function rollTurnOrder():Void { if (rolling == false) { ran = Std.random(6); Registry.randomBattleAnim.animation.play("turnorder"); rolling = true; haxe.Timer.delay(endAnimation, 2000); } } private function endAnimation():Void { Registry.randomBattleAnim.animation.gotoAndStop(ran); rolling = false; }

Any suggestions would be welcome.

All I can offer is an alternative solution.

Since animations names are stored as string values, you could try naming your roll result animations "roll1"/"roll2"/etc and have the animations played at the end of the generic rolling animation through an animation callback or timer as above.

Example:

The die is rolled. As the generic roll animation begins, the result is decided to be a 2. When the generic roll ends (reaches a point of transition to the result animation), tell the die to play("roll" + result). It will play the result animation for rolling a 2 given you named that animation "roll2". This way you don't have to worry about frame numbers etc aside from when you create the animation initially.

And I found the solution. I had to dig deeper into the animation stuff to figure it out. But to do a goToAndStop equivolent in 3.0.0 you have to do these two lines:

Registry.randomBattleAnim.animation.curAnim.stop(); Registry.randomBattleAnim.animation.curAnim.curFrame = ran;

Not as simple, but works the way it should.
