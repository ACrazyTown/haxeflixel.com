Hello again, Peko! I find another comment crazysam made on nape forum which counts more.

Nape is not currently totally ready for openfl because there are flags for nme but not for openfl in nape itself.(Only the debug shape part is not ready as I understand)

Nape's creator recommends using nme instead of openfl.

You can install nme with haxelib install nme and set <haxelib name="nme" />​ instead of openfl in your nmml/xml file. Works for linux in my test and neko still throws​ the std@module_read​ exception.

Or you can work on flash target now with openfl since it's working and wait until some changes are made inside nape.

I was having the exact same problem of

Uncaught exception - std@module_read

But then with the combination of the latest versions of Haxe 3.0.1 and Neko 2.0.0, as well as putting this in my project.xml

<haxedef name="NAPE_RELEASE_BUILD"/>

and using the latest dev versions of hxcpp, nme, nmedev, openfl, openfl-native, and openfl-tools I am now successfully running Nape samples on Neko.

Things are just a bit unstable at the moment ;)
